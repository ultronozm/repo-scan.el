#+title: repo-scan.el: Dired-like dashboard for Git repo collections
#+author: Paul Nelson

=repo-scan= is a small Emacs package for managing a personal collection of
Git repositories from one buffer.

The main command is =M-x repo-scan=.  It reads configured repository sources,
starts asynchronous Git-state scans for each repository, and displays a
tabulated dashboard with dirty, ahead, behind, missing, stale-branch, and
remote-mismatch information as rows finish.

* Manifest format

Customize =repo-scan-manifest-files= to specify your manifest files, e.g.:
#+begin_src elisp
(setopt repo-scan-manifest-files '("~/dotfiles/repos.manifest"))
#+end_src

Each non-comment line of each manifest file should begin with a path and origin URL.  Blank lines and lines starting with =#= are ignored.  A third field may specify an extra remote as =NAME=URL=.  Additional fields can set scan policy, such as =sync=wip= or =expected-local=working=.  Extra fields are ignored.  Example:
#+begin_example
~/work/example git@github.com:user/example.git
~/work/fork git@github.com:user/fork.git upstream=https://github.com/upstream/fork.git
~/work/emacs git://git.sv.gnu.org/emacs.git wip=git@github.com:user/emacs-wip.git expected-local=working
#+end_example

* Customization

| Variable | Purpose |
|----------+---------|
| =repo-scan-manifest-files= | Manifest files read by =repo-scan-source-manifests= |
| =repo-scan-repos= | Repository base names read by =repo-scan-source-registered= |
| =repo-scan-base-directory= | Base directory for =repo-scan-repos= |
| =repo-scan-sources= | Source functions that return repository descriptors |
| =repo-scan-extra-roots= | One-level Git repository roots for the extra-roots source |
| =repo-scan-emacs-package-roots= | One-level package clone roots for the Emacs package source |
| =repo-scan-refresh-concurrency= | Maximum number of async scan subprocesses |
| =repo-scan-git-program= | Git executable |
| =repo-scan-fetch-arguments= | Git arguments used by =repo-scan-fetch= |
| =repo-scan-sync-policies= | Per-repository sync policy for safe actions |
| =repo-scan-expected-local-branches= | Per-repository generated/local branches omitted from local-only counts |
| =repo-scan-buffer-name= | Dashboard buffer name |
| =repo-scan-command-buffer-name= | Shell command output buffer name |
| =repo-scan-diff-buffer-name= | Diff buffer name |
| =repo-scan-log-buffer-name= | Log buffer name |

* Example configuration

Here is a complete example config using [[https://github.com/progfolio/elpaca][elpaca]]:

#+begin_src emacs-lisp
(use-package repo-scan
  :ensure (:host github :repo "ultronozm/repo-scan.el")
  :defer t
  :custom
  (repo-scan-manifest-files '("~/dotfiles/repos.manifest")))
#+end_src

If you want packages declared with =use-package= to register themselves in
=repo-scan-repos= at startup, define the following local =:repo-scan= keyword
before those declarations are expanded.  This is the recommended integration
when your package declarations already name the repositories you want the
dashboard to track, and those names match clone directories under
=repo-scan-base-directory=, as they do with Elpaca's default layout:

#+begin_src emacs-lisp
(defalias 'use-package-normalize/:repo-scan
  'use-package-normalize-predicate)

(defun use-package-handler/:repo-scan (name _keyword pred rest state)
  "Handle :repo-scan keyword in `use-package' forms."
  (use-package-concat
   (when pred
     `((with-eval-after-load 'repo-scan
         (add-to-list 'repo-scan-repos ,(symbol-name name)))))
   (use-package-process-keywords name rest state)))

(setq use-package-keywords
      (use-package-list-insert
       :repo-scan (remove :repo-scan use-package-keywords) :after))
#+end_src

The keyword is inserted before =:after= so that =:after= does not delay
registration.  With this in place, a package declaration such as:

#+begin_src emacs-lisp
(use-package my-package
  :repo-scan
  :ensure (:host github :repo "user/my-package.el"))
#+end_src

adds ="my-package"= to =repo-scan-repos= once =repo-scan= loads.

* Usage

In the dashboard:

| Key   | Command |
|-------+---------|
| =n=, =p= | Move to next/previous repository |
| =g= | Refresh all repository state |
| =.= | Rescan the repository at point, or all marked repos |
| =RET= | Open =vc-dir= for the repository |
| =i= | Explain the row state with reproduction commands and next steps |
| =j= | Open Dired at the repository root |
| =A= | Search tracked files in marked repos, or the repo at point |
| =C-x g= | Open Magit status for the repository, when Magit is available |
| ~=~ | Show a useful diff or local-only commit patches for the repository |
| =l= | Show a compact Git log for the current, selected drift, or local-only branch |
| =f= | Fetch all remotes for the current repository |
| =F= | Fetch all remotes for all known repositories asynchronously |
| =+= | Open/reuse =vc-dir= for the repository and call =vc-pull= |
| =P= | Open/reuse =vc-dir= for the repository and call =vc-push= |
| =m=, =u=, =t=, =U= | Mark, unmark, toggle, or clear marks |
| =x= | Run the default safe action on marked repos, or the current repo |
| =X= | Run the default safe action on every eligible repo |
| =!= | Run a shell command in marked repos, or the current repo |
| =&= | Run an async shell command in marked repos, or the current repo |
| =?= | Describe the dashboard mode and bindings |

Prefix arguments are passed along, so =C-u += and =C-u P= behave like prefix calls to those VC commands.  VC commands may run asynchronously; press =g= when they finish to refresh the dashboard.

The default safe bulk action is intentionally narrow: =git pull --ff-only= on a clean current branch whose upstream is behind and not ahead.  It does not push, delete branches, merge divergent branches, or touch dirty working trees.

* Mark helpers

The =*= prefix marks repositories by state:

| Key | Marks |
|-----+-------|
| =* p= | Clean repos that can be safely pulled |
| =* P= | Clean repos that can be safely pushed |
| =* d= | Dirty repos |
| =* b= | Repos with any local branch behind an upstream |
| =* a= | Repos with any local branch ahead of an upstream |
| =* s= | Repos with stale non-current local branches |

* Repository sources

The dashboard is provider-based.  By default, =repo-scan-sources= includes
=repo-scan-source-manifests= followed by =repo-scan-source-registered=.

=repo-scan-source-manifests= reads the files listed in
=repo-scan-manifest-files= using the manifest format above.

=repo-scan-source-registered= returns descriptors for each base name in
=repo-scan-repos=, expanded relative to =repo-scan-base-directory=.  Its
default base directory is:

#+begin_src emacs-lisp
(expand-file-name "elpaca/repos/" user-emacs-directory)
#+end_src

For example:

#+begin_src emacs-lisp
(setopt repo-scan-repos '("magit" "consult"))
#+end_src

* Emacs package repositories

To scan package repositories, set =repo-scan-emacs-package-roots= to
one-level package clone roots and add the source:

#+begin_src emacs-lisp
(setopt repo-scan-emacs-package-roots
        (list (expand-file-name "elpaca/repos" user-emacs-directory)))
(add-to-list 'repo-scan-sources
             'repo-scan-source-emacs-package-roots
             t)
#+end_src

Or open only that source with:

#+begin_src emacs-lisp
(repo-scan-emacs-packages)
#+end_src

For other one-level roots, customize =repo-scan-extra-roots= and add
=repo-scan-source-extra-roots= to =repo-scan-sources=.

Large package roots can contain hundreds of Git repositories.  Keep this source
opt-in if you mostly want the fast manifest view.
