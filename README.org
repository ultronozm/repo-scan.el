#+title: repo-dashboard.el: Dired-like dashboard for Git repo manifests
#+author: Paul Nelson

=repo-dashboard= is a small Emacs package for managing a personal collection of
Git repositories from one buffer.  It is meant to sit between a shell script
such as =install --doctor= and heavier per-repository tools such as =vc-dir= or
Magit.

The main command is =M-x repo-dashboard=.  It reads one or more manifest files,
starts asynchronous Git-state scans for each repository, and displays a
tabulated dashboard with dirty, ahead, behind, missing, stale-branch, and
remote-mismatch information as rows finish.

* Manifest format

The default manifest is =~/dotfiles/repos.manifest=.  Each non-comment line
begins with a path and origin URL:

#+begin_example
~/work/example git@github.com:user/example.git
~/work/fork git@github.com:user/fork.git upstream=https://github.com/upstream/fork.git
#+end_example

Blank lines and lines starting with =#= are ignored.  A third field may specify
an extra remote as =NAME=URL=.  Extra fields are ignored, matching the simple
manifest style used by shell setup scripts.

When an origin or extra remote is present, =repo-dashboard= checks that the
local remote URL matches.  SSH, HTTPS, and =.git= suffix differences are
normalized before comparison.

Customize =repo-dashboard-manifest-files= to read different manifests.

* Usage

#+begin_src emacs-lisp
(use-package repo-dashboard
  :load-path "~/work/repo-dashboard"
  :bind (("C-c r" . repo-dashboard)))
#+end_src

In the dashboard:

| Key   | Command |
|-------+---------|
| =n=, =p= | Move to next/previous repository |
| =g= | Refresh repository state asynchronously |
| =C-u g= | Refresh repository state synchronously |
| =RET= | Open =vc-dir= for the repository |
| =j= | Open Dired at the repository root |
| =C-x g= | Open Magit status for the repository, when Magit is available |
| === | Show a useful diff for the repository |
| =l= | Show a compact Git log for the current or selected drift branch |
| =f= | Fetch all remotes for the current repository |
| =F= | Fetch all remotes for all known repositories asynchronously |
| =+= | Open/reuse =vc-dir= for the repository and call =vc-pull= |
| =P= | Open/reuse =vc-dir= for the repository and call =vc-push= |
| =m=, =u=, =t=, =U= | Mark, unmark, toggle, or clear marks |
| =x= | Run the default safe action on marked repos, or the current repo |
| =X= | Run the default safe action on every eligible repo |
| =!= | Run a shell command in marked repos, or the current repo |
| =&= | Run an async shell command in marked repos, or the current repo |
| =?= | Describe the dashboard mode and bindings |

The individual pull and push commands are wrappers around VC, but they run from
a real =vc-dir= buffer for the repository.  This matches the context required
by =vc-pull= and =vc-push=.  Prefix arguments are passed along, so =C-u += and
=C-u P= behave like prefix calls to those VC commands.  VC commands may run
asynchronously; press =g= when they finish to refresh the dashboard.

The default safe bulk action is intentionally narrow: =git pull --ff-only= on a
clean current branch whose upstream is behind and not ahead.  It does not push,
delete branches, merge divergent branches, or touch dirty working trees.

* Async refresh

Dashboard refreshes are asynchronous by default.  A refresh renders
=scanning= placeholder rows immediately, then fills in each row as a background
Emacs subprocess completes the existing synchronous repository scanner.  This
keeps large manifests and package roots responsive while preserving the same
Git-state interpretation as the synchronous path.

Customize =repo-dashboard-refresh-concurrency= to control how many scan
subprocesses run at once.  Set =repo-dashboard-async-refresh= to nil, or press
=C-u g= in the dashboard, to force the old blocking refresh behavior.

* Mark helpers

The =*= prefix marks repositories by state:

| Key | Marks |
|-----+-------|
| =* p= | Clean repos that can be safely pulled |
| =* P= | Clean repos that can be safely pushed |
| =* d= | Dirty repos |
| =* b= | Repos with any local branch behind an upstream |
| =* a= | Repos with any local branch ahead of an upstream |
| =* s= | Repos with stale non-current local branches |

* Stale deploy branches

=repo-dashboard= treats local branches named =deploy= as ordinary local
branches, but it makes common generated-branch states visible:

- =stale deploy=: local =deploy= has no unique commits and is behind
  =origin/deploy=.
- =deploy diverged=: local =deploy= and =origin/deploy= both have unique
  commits.

Remote-only deploy branches, such as a repository with =main= plus
=origin/deploy= but no local =deploy=, do not count as a problem.

The package does not delete or reset these branches automatically.  Use =RET= to
enter =vc-dir=, or =C-x g= for Magit, when you want to perform branch surgery.

* Emacs package repositories

The dashboard is provider-based.  The default provider reads manifest files, but
you can include other sources.

To scan package repositories under =~/.emacs.d/elpaca/repos=:

#+begin_src emacs-lisp
(add-to-list 'repo-dashboard-sources
             'repo-dashboard-source-emacs-package-roots
             t)
#+end_src

Or open only that source with:

#+begin_src emacs-lisp
(repo-dashboard-emacs-packages)
#+end_src

For other one-level roots, customize =repo-dashboard-extra-roots= and add
=repo-dashboard-source-extra-roots= to =repo-dashboard-sources=.

Large package roots can contain hundreds of Git repositories.  Keep this source
opt-in if you mostly want the fast manifest view, and lower
=repo-dashboard-refresh-concurrency= if scanning too many packages at once is
too noisy for your machine.

* Notes

This package deliberately stays close to built-in VC:

- =RET= goes to =vc-dir= rather than Magit.
- =+= and =P= call =vc-pull= and =vc-push= when those functions are available.
- =C-x g= is a local convenience binding for Magit users.

The goal is not to replace per-repository tools.  It is to make it easy to see
which repositories need attention, mark a subset, and perform conservative
bulk actions without leaving Emacs.
